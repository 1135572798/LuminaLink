# LuminaLink Release Guide

- Current version: `0.1.3`
- Repository: `https://github.com/1135572798/LuminaLink`
- Release channel: GitHub Releases

## Artifact Types

LuminaLink publishes two Windows artifacts:

```text
LuminaLink-Setup-0.1.3-x64.exe       Installer
LuminaLink-Portable-0.1.3-x64.exe    Portable app
```

The source ZIP generated by GitHub is only the source code. Normal users should
download one of the `.exe` files from the Releases page.

## Build Locally

Install dependencies:

```powershell
pnpm install
```

Build the app and package Windows artifacts:

```powershell
pnpm dist:win
```

Output path:

```text
release/0.1.3/
```

## Installer Behavior

The installer target uses NSIS:

- interactive install wizard
- per-user install by default
- custom install directory allowed
- Start Menu shortcut enabled
- desktop shortcut enabled
- local user data is preserved on uninstall

Local data remains in the standard LuminaLink app data folders:

```text
%APPDATA%/LuminaLink/config.json
%LOCALAPPDATA%/LuminaLink/luminalink.sqlite
%LOCALAPPDATA%/LuminaLink/translation-cache.sqlite
```

## Release Checklist

```powershell
pnpm build
pnpm dist:win
pnpm luminalink doctor --json
git status --short
```

Then upload these files to GitHub Releases:

```text
release/0.1.3/LuminaLink-Setup-0.1.3-x64.exe
release/0.1.3/LuminaLink-Portable-0.1.3-x64.exe
```

Do not upload local SQLite databases, `.env` files, API keys, tokens, cookies,
or private keys.
