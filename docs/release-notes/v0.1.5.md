# LuminaLink v0.1.5

This patch fixes translation failures that were hard to diagnose on Windows.

## Changes

- Honor `HTTPS_PROXY`, `HTTP_PROXY`, `ALL_PROXY`, and `NO_PROXY` for OpenAI translation requests.
- Run a real tiny translation probe from "жЈЂжџҐзЋЇеўѓ" instead of only checking whether a key exists.
- Show friendly Chinese messages for common OpenAI failures, including `429 insufficient_quota`.
- Persist the last translation failure reason and show it in the asset detail pane.
- Keep failed/translated status when a scan sees unchanged asset content.
- Rename the sidebar Provider label from "Provider зЉ¶жЂЃ" to "Provider й…ЌзЅ®".
